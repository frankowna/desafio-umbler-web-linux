# Pergunta 1 - Linux e Troubleshooting

## Objetivo

Esta seção apresenta minha abordagem para investigação e resolução de problemas em ambientes Linux de produção.

O foco está na identificação da causa raiz através da análise de serviços, recursos do sistema, logs e conectividade, evitando alterações precipitadas no ambiente.

## Arquivos

- respostas.md → Respostas detalhadas do cenário proposto.
- health_check.sh → Script simples para validação rápida da saúde do ambiente.

## Premissas

Durante o troubleshooting procuro seguir uma sequência lógica:

1. Validar disponibilidade dos serviços.
2. Analisar logs.
3. Verificar utilização de recursos.
4. Confirmar conectividade.
5. Identificar a causa raiz.
6. Aplicar a correção.
7. Validar o funcionamento após a intervenção.
